# 00 — Dossier de référence

## Dossier principal

Le dossier de référence principal du projet est : `/docs`.

C'est la source de vérité pour :

- la vision produit ;
- l'architecture ;
- les spécifications backend/frontend ;
- les règles de développement ;
- les guides d'intégration (Supervision, modèles, device) ;
- les checklists de contribution.

## Sous-dossier recommandé

Pour rassembler les ressources externes, exemples, et références longues, utiliser :

```
/docs/reference/
```

## Fichier de suivi des contributions

Créer et maintenir :

```
/docs/CONTRIBUTIONS.md
```

Format recommandé (template) :

- Date — Auteur — Brève description
- Fichiers modifiés :
  - [path/to/file]
- Commandes exécutées :
  - `...`
- Checklist :
  - [ ] Action restante 1
  - [x] Action complétée 2
- Notes / next steps

Chaque contributeur doit mettre à jour ce fichier (ou `CLAUDE.md`) lors des interventions importantes.
